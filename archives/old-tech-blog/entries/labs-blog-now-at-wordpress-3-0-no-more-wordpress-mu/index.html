Labs blog now at WordPress 3.0; no more WordPress-mu

by nkinkade on 2010-07-02 none

For quite a few years CC has maintained an installation of WordPress-mu, which has hosted this blog, as well as blogs for a couple of CC jurisdictions. Since CC's main site is built on WordPress, this meant that with every new release of WordPress we would have to upgrade the main site, and also our WordPress-mu install. Upgrading the main CC site was/is very easy, because it is managed with version control.

Well, with the release of WordPress 3.0, the functionality of WordPress-mu was integrated into the core WordPress code, so it's no longer necessary to maintain two separate code bases. Upgrading Labs to WordPress 3.0 was (mostly) as easy as checking out the code that runs creativecommons.org and dropping our WordPress-mu wp-config.php file into it, then running the update.php script. We now maintain a single WordPress code base that runs both creativecommons.org, as well as our multi-site installation, which hosts this blog.
